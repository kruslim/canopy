# `data/` — provenance ledger

This directory holds signal-decode dictionaries and captured logs. Its existence is
governed by one non-negotiable rule from the project charter (`docs/00-charter-and-index.md`):

> **Never** commit real proprietary CAN databases, capture files, internal signal
> definitions, or test data. Use standard OBD-II PIDs, **open** DBC files, and
> **synthetic or self-captured** logs only.

If a reviewer asks *"is any of this proprietary?"* the answer must be an immediate,
confident **no** — and this file is the artifact that proves it.

## Directory contract

| Path | Allowed contents | Provenance |
|------|------------------|------------|
| `data/dbc/` | **Open DBC files only** (e.g. the OpenDBC project). | Public / open-source. |
| `data/captures/` | **Synthetic** or **self-captured** logs from your own vehicle only. | Generated by `SyntheticReader`, or your own hardware. |

Anything proprietary belongs under `data/captures/proprietary/` — which is `.gitignore`d
and must never be committed. `*.private.dbc` is likewise ignored.

## Current status — Phase 0

Both `dbc/` and `captures/` are **empty** (each keeps a `.gitkeep` placeholder). Phase 0
runs entirely on deterministic, in-memory synthetic data produced by
`src/canopy/readers/synthetic.py`; no files are read from disk. Real open DBCs arrive in
Phase 5 (`docs/08-raw-can-extension.md`).

## The distinction that makes this project showable

Domain **expertise** is yours and portable. Domain **artifacts** belong to your employer.
Every file added here must have a provenance line in the table above before it is committed.
